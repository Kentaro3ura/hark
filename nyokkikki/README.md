
# Hark Hark にょっきっきプログラムについて

たけのこにょっきっきのゲーム進行と審判を行うプログラムです。
Harkの音源定位機能を使って、同時発話と最後に残った人(発話できなかった人)を検知します。

本プログラムは、2017年12月6日のHarkハッカソンで作成しました。


# 実行環境

* Harkがインストールされていること
* PATH環境変数にpython、batchflowのパスが通っていること


# 使用方法

* PCにマイクアレイのたまごを接続し、基準(黒い点)がPC(スピーカー)と向かい合うように設置します。
* ５人の参加者をたまごの周りに等間隔で配置します。
* PCから時計周りで１番、２番、…、５番とラベリングされます。
* game_start.batを実行(ダブルクリック)します。
* ゲームが開始されます。
* 本プログラムは、以下を検知します。
  - フライング(スピーカーとの重複発話で検知)
  - プレーヤー同士の重複発話
  - 最後まで発話しなかったプレーヤー

* 失敗したプレーヤーをアナウンスしてプログラムが終了します。


# 注意事項

本ゲームで遊ぶには、遊ぶ環境に合わせてHarkのチューニングが必要です。
Harkの音の感度(THRES)が低すぎるとプレーヤーの発話を検知せず、逆に高すぎると誤検知しやすくなります。


# 課題

* 声が小さい場合などにプログラムがプレーヤーの発話を正しく認識しないことがある。プログラムが発話を認識したかどうかをフィードバックする仕組み(例えば音を鳴らすとか)が欲しい。
* 参加人数は５名固定だが、可変にしたい。
* 音声認識機能を使って正しく発話されたかを判定できるようにし、正しく発話した場合のみクリアと判断するようにしたい。
